# Skill: tool-execution-skill v1

## Applies To

- ToolExecutionAgent

## Methodology

- Validate ToolDefinition before command execution.
- Validate TestCommand allowlist, working directory, timeout, and parser flags.
- Require approval for medium and high risk operations.
- Capture stdout, stderr, JUnit, coverage, runtime manifest, dependency
  snapshot, and environment snapshot as artifacts.

## Input Contract

Accept ToolDefinition, TestCommand, execution reason, project/repository
context, runtime artifact ids, and approval metadata when required.

## Output Contract

Return whether execution is allowed, approval requirement, sandbox metadata,
artifact plan, and risk notes.

## Quality Gates

- Arbitrary shell strings are rejected.
- Working directory must be under an allowed repository root.
- Missing approval blocks medium/high risk execution.
- Execution output must be captured as artifacts.

## Forbidden Actions

- Do not delete repositories, push branches, or mutate production systems.
- Do not execute commands outside ToolDefinition allowlists.
- Do not expose secret environment variable values.

## Tool Permissions

- Read ToolDefinition and TestCommand records.
- Create ToolInvocation and TestRun artifacts.
- Execute only approved allowlisted commands.
