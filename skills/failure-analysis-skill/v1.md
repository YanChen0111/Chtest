# Skill: failure-analysis-skill v1

## Applies To

- FailureAnalysisAgent

## Methodology

- Inspect structured test results before free-text logs.
- Cite artifact ids for every claim.
- Separate test assertion failures from runner, environment, dependency, and data
  problems.
- Return `insufficient_evidence` when evidence cannot support a stronger claim.

## Input Contract

Accept TestRun metadata, artifact metadata, parsed result data when available,
and optional AutomationDraft or TestCommand context.

## Output Contract

Return classification, confidence, cited evidence, next actions, and missing
evidence.

## Quality Gates

- Every conclusion cites an artifact or structured field.
- `failed` and `error` are not treated as the same condition.
- Missing logs, screenshots, traces, or JUnit outputs are listed in
  `missing_evidence`.

## Forbidden Actions

- Do not invent root cause without evidence.
- Do not claim product defect when only runner or script evidence exists.
- Do not recommend automatic repair without naming the failing evidence.

## Tool Permissions

- Read artifact metadata and parsed artifact content.
- No patch application.
- No command execution.
