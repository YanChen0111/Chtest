# Skill: test-design-skill v1

## Applies To

- TestDesignAgent

## Methodology

- Apply equivalence partitioning, boundary value analysis, state transition
  testing, decision-table thinking, negative testing, exploratory risk notes,
  regression design, and security abuse-case prompts when supported.
- Use TestKnowledgeCard and KnowledgeEvidence to choose design techniques.
- Prefer small reviewable test design notes over broad generic strategy text.

## Input Contract

Accept requirements, risk items, approved TestKnowledgeCards, KnowledgeEvidence,
existing case summaries, and coverage gaps. Inputs may include execution
evidence summaries when regression or failure-driven design is needed.

## Output Contract

Return test design notes, selected techniques, target coverage areas, candidate
case type hints, automation fit hints, and review findings. Output is guidance
for CaseGenerationAgent and CaseReviewAgent, not executable code.

## Quality Gates

- Every design recommendation cites requirement, risk, or KnowledgeEvidence.
- Boundary and exception techniques are required for P0/P1 flows when supported.
- Historical defect evidence produces regression design notes.
- Unsupported design ideas are listed as coverage gap notes.
- Design output must be concise enough for human review.

## Forbidden Actions

- Do not call external providers.
- Do not create vector indexes.
- Do not trigger MCP runtime.
- Do not promote generated cases.
- Do not generate automation code or ToolInvocations.
- Do not claim coverage without evidence.

## Tool Permissions

- Read supplied evidence only.
- No execution tools.
- No write tools outside design artifact output.
- No MCP tools.
