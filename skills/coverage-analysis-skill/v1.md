# Skill: coverage-analysis-skill v1

## Applies To

- CoverageAnalysisAgent

## Methodology

- Compare requirements, risks, TestKnowledgeCard summaries, KnowledgeEvidence,
  existing cases, and execution evidence.
- Identify missing requirement, risk, boundary, exception, data, security,
  regression, and historical-defect coverage.
- Use GraphRAG-style relationship thinking only over supplied inputs.
- Preserve coverage gaps as review evidence instead of creating hidden tasks.

## Input Contract

Accept requirement summaries, risk items, existing case summaries, execution
evidence summaries, approved TestKnowledgeCards, and normalized
KnowledgeEvidence. Inputs must already be local and prompt-eligible.

## Output Contract

Return coverage gap notes, covered requirement ids, covered risk ids, duplicate
risk hints, missing evidence notes, and review findings suitable for case review
or test design.

## Quality Gates

- Every coverage gap references a requirement, risk, case, or KnowledgeEvidence
  item.
- Missing evidence is explicit and never replaced with guessed coverage.
- P0/P1 gaps are clearly separated from lower-priority gaps.
- Duplicate risks preserve distinct risk coverage when cases overlap.
- Coverage findings remain review evidence until a human accepts them.

## Forbidden Actions

- Do not call external providers.
- Do not create vector indexes.
- Do not trigger MCP runtime.
- Do not promote generated cases.
- Do not run graph extraction or create graph jobs.
- Do not create, update, or delete TestCase records.

## Tool Permissions

- Read supplied summaries and artifacts only.
- No execution tools.
- No KnowledgeAdapter provider calls.
- No MCP tools.
