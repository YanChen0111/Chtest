# Skill: risk-analysis-skill v1

## Applies To

- RiskAnalysisAgent

## Methodology

- Classify business, data, environment, security, technical, and regression
  risks.
- Use requirement acceptance points, constraints, TestKnowledgeCard summaries,
  KnowledgeEvidence, historical failures, and existing case gaps.
- Prioritize P0/P1 risks when user harm, financial impact, data integrity,
  security, or release-blocking regression is plausible.
- Keep unsupported risk ideas as review findings, not facts.

## Input Contract

Accept requirement understanding output, requirement text, risk taxonomy,
approved TestKnowledgeCards, normalized KnowledgeEvidence, historical failure
summaries, and existing case summaries. Inputs must already be local and
prompt-eligible.

## Output Contract

Return draft risk items with title, risk level, category, impact, test
suggestion, source KnowledgeEvidence ids, unsupported claims, and review
findings. Output remains review evidence until a human accepts it.

## Quality Gates

- Every P0/P1 risk cites KnowledgeEvidence or records missing evidence.
- Each risk has a concrete test suggestion.
- Regression risks cite historical failure, defect, case, or report evidence
  when available.
- Security and data risks are not invented when no evidence supports them.
- Risk analysis must not create or approve TestCase records.

## Forbidden Actions

- Do not call external providers.
- Do not create vector indexes.
- Do not trigger MCP runtime.
- Do not promote generated cases.
- Do not create persisted RiskItem, TestCase, TestRun, Report, retrieval,
  vector, or graph records.

## Tool Permissions

- Read supplied evidence only.
- No execution tools.
- No KnowledgeAdapter provider calls.
- No MCP tools.
