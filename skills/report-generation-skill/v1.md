# Skill: report-generation-skill v1

## Applies To

- ReportAgent

## Methodology

- Build reports from structured evidence before narrative text.
- Cite artifacts for every conclusion.
- Separate passed, failed, needs_attention, and insufficient_evidence.
- Preserve enough details for a later reviewer to reproduce the conclusion.

## Input Contract

Accept report type, related entity metadata, evidence artifact metadata,
structured metrics, optional QualityGateDecision, optional TestRun summaries,
and optional FailureAnalysis records.

## Output Contract

Return conclusion, summary, metrics, evidence artifact ids, next actions, and
renderable report content for Markdown, HTML, or JSON outputs.

## Quality Gates

- Every conclusion cites artifact ids or structured metrics.
- Missing evidence cannot produce a passed conclusion.
- CI/CD quality reports align with the latest QualityGateDecision.

## Forbidden Actions

- Do not invent artifacts, test results, screenshots, traces, or logs.
- Do not claim deployment or merge status in V1.
- Do not override QualityGateDecision without evidence.

## Tool Permissions

- Read artifact metadata and parsed artifact content.
- Write report artifacts.
- No test execution or patch application.
