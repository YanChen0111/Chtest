# Skill: knowledge-ingestion-skill v1

## Applies To

- KnowledgeIngestionAgent

## Methodology

- Extract small TestKnowledgeCard candidates from source text.
- Preserve source artifact id, source span, quote, hash, and section path.
- Prefer PageIndex-style tree and section traceability for long documents.
- Separate requirement points, business rules, API contracts, boundary
  conditions, exception scenarios, risk points, bug patterns, existing case
  patterns, anti-patterns, and test strategy notes.
- Mark uncertainty and conflict as review findings.

## Input Contract

Accept prompt-eligible ContextArtifacts, reviewed cases, failure summaries,
reports, and optional existing TestKnowledgeCard summaries. Inputs may include
KnowledgeEvidence for prior context but must not require external provider calls.

## Output Contract

Return draft TestKnowledgeCard candidates with source artifact, source span,
source quote or hash, module key, risk type, case type hint, confidence,
safe_to_show, allowed_for_prompt, and review findings.

## Quality Gates

- Every card cites a source artifact and stable source span or hash.
- Every card is specific enough to guide case generation or review.
- Unsafe or non-prompt-eligible source content is not copied into prompt output.
- Duplicate, conflicting, or low-confidence cards are marked for human review.
- KnowledgeEvidence references remain normalized Chtest evidence.

## Forbidden Actions

- Do not call external providers.
- Do not create vector indexes.
- Do not trigger MCP runtime.
- Do not promote generated cases.
- Do not mark draft cards as approved or prompt-eligible.
- Do not create retrieval jobs, graph jobs, TestCase records, TestRun records,
  reports, or provider configs.

## Tool Permissions

- Read supplied artifact content only.
- No execution tools.
- No write tools outside draft artifact output.
- KnowledgeAdapter provider calls are not allowed in this seed.
