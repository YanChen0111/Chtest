# Skill: requirement-review-skill v1

## Applies To

- RequirementReviewAgent

## Methodology

- Completeness review.
- Clarity review.
- Consistency review.
- Testability review.
- Feasibility review.
- Logic review.
- Risk-oriented test design.

## Input Contract

Accept one requirement body, optional module name, optional target test types,
and optional ContextArtifact manifest entries.

## Output Contract

Return six scores, issue list, clarification questions, test design notes,
`used_knowledge`, and `used_context_artifact_ids`.

## Quality Gates

- All six dimensions must be present.
- Every issue must cite evidence from the requirement or supplied context.
- The result must distinguish local ContextArtifacts from external knowledge.
- At least one test design note is required when the requirement is reviewable.

## Forbidden Actions

- Do not claim external RAG evidence when `use_knowledge=false`.
- Do not silently ignore supplied ContextArtifact ids.
- Do not promote cases or automation drafts.

## Tool Permissions

- KnowledgeAdapter.search_context optional only when `use_knowledge=true`.
- No execution tools.
