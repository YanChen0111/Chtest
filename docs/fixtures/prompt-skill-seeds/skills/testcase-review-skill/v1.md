# Skill: testcase-review-skill v1

## Applies To

- CaseReviewAgent

## Methodology

- Check requirement traceability first.
- Identify missing positive, negative, boundary, and state transition coverage.
- Compare with existing cases to avoid duplicate coverage.
- Recommend edits instead of silently approving weak cases.

## Input Contract

Accept requirement evidence, generated candidate cases, optional risk items, and
optional existing case summaries.

## Output Contract

Return per-case review results, issue reasons, optimization suggestions, and
whether each candidate is ready for human approval.

## Quality Gates

- Every approved recommendation must cite requirement or risk evidence.
- Missing expected result, precondition, or executable step blocks approval.
- Duplicate coverage must be marked for review or merge.

## Forbidden Actions

- Do not create official TestCase records.
- Do not approve vague cases.
- Do not invent requirement details.

## Tool Permissions

- Read candidate case and requirement artifacts.
- No execution tools.
- No repository writes.
