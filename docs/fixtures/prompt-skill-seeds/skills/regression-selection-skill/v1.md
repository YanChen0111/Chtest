# Skill: regression-selection-skill v1

## Applies To

- CICDChangeAnalysisAgent
- RegressionAgent

## Methodology

- Classify changed files by role before recommending tests.
- Prefer direct unit coverage for changed source branches.
- Select the smallest regression set that still covers impacted modules.
- Escalate to `needs_review` when risk evidence is incomplete.

## Input Contract

Accept CICDRun metadata, changed file records, risk analysis artifact content,
available TestCommand records, and optional historical TestRun summaries.

## Output Contract

Return risk reasons, impacted areas, recommended tests or commands, and evidence
references. For CI/CD change analysis, output must be serializable as
`risk_analysis.json`. For regression selection, every recommended command must
include a reason and expected signal.

## Quality Gates

- Every recommendation cites a changed file, risk reason, or prior test signal.
- High-risk source changes require either a direct UnitTestPatch recommendation
  or an explicit `needs_review` reason.
- Regression recommendations must not require network access in V1 unless a
  TestCommand explicitly allows it.

## Forbidden Actions

- Do not invent changed files, branches, CI providers, or PR metadata.
- Do not recommend modifying business source files.
- Do not mark the quality gate as passed when tests or artifacts are missing.

## Tool Permissions

- Read diff, changed file, artifact, and TestCommand metadata.
- No patch application.
- No deployment, merge, push, or remote CI actions.
