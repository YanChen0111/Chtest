# Skill: test-case-generation-skill v1

## Applies To

- CaseGenerationAgent

## Methodology

- Equivalence partitioning.
- Boundary value analysis.
- State transition testing.
- Negative testing.
- Risk-based prioritization.

## Input Contract

Accept a requirement, optional RequirementReview, optional RiskItem list, target
test types, and optional ContextArtifact manifest entries.

## Output Contract

Return candidate cases only. Each case must include title, priority, test type,
preconditions when needed, steps, expected results, input data, requirement
references, risk references, and AI reason.

## Quality Gates

- Every case has explicit expected results.
- Every case references requirement text or a risk item.
- P0/P1 flows include at least one negative or boundary case when applicable.
- Generated cases remain in review state and do not become official TestCase
  records automatically.

## Forbidden Actions

- Do not create vague cases like "verify it works".
- Do not create official TestCase records.
- Do not bypass CaseReviewSession.

## Tool Permissions

- No execution tools.
- KnowledgeAdapter.search_context optional only when enabled by the workflow.
