# Skill: automation-draft-skill v1

## Applies To

- AutomationDraftAgent

## Methodology

- Translate reviewed behavior into executable test intent.
- Prefer small pytest or Playwright drafts with explicit assumptions.
- Use existing command and repository context when supplied.
- Keep generated code reviewable before execution.

## Input Contract

Accept one Requirement or approved TestCase, target framework, project language,
optional repository context, optional TestCommand, and optional ContextArtifact
manifest entries.

## Output Contract

Return AutomationDraft candidates with target framework, title, suggested file
path, draft language, draft code, execution notes, risk notes, and used context
artifact ids.

## Quality Gates

- Draft must include `target_framework`, `suggested_file_path`, and `draft_code`.
- Draft must name fixture, data, selector, dependency, or environment assumptions.
- Draft must be suitable for review before execution.
- Draft must not modify business source files.

## Forbidden Actions

- Do not apply generated code to the target repository.
- Do not execute the draft.
- Do not hide risky assumptions.
- Do not add skip or xfail to conceal failures.

## Tool Permissions

- No execution tools during generation.
- TestRunnerTool may be used only after approval in a later workflow step.
