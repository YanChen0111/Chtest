# Skill: unit-test-generation-skill v1

## Applies To

- UnitTestAgent

## Methodology

- Read changed source behavior before creating tests.
- Prefer boundary value and branch coverage for changed logic.
- Keep the generated patch small and reviewable.
- Run PatchScopeGate before any application.

## Input Contract

Accept CICDRun metadata, CICDChangedFile records, diff artifacts, risk analysis
artifact content, target framework, and optional existing test snippets.

## Output Contract

Return a UnitTestPatch candidate with unified diff, target framework, test
intent, coverage target, and risk notes.

## Quality Gates

- Patch modifies only allowed test directories.
- Patch covers at least one changed behavior or high-risk branch.
- Patch does not add skip, xfail, broad no-op assertions, or source edits.

## Forbidden Actions

- Do not modify business source files, migrations, lockfiles, or runtime config.
- Do not hide failures by weakening assertions.
- Do not apply the patch without human approval.

## Tool Permissions

- Read diff and test context artifacts.
- Write UnitTestPatch artifact.
- No direct repository writes.
