# Prompt: failure_analysis v1

## Agent

FailureAnalysisAgent

## Purpose

Classify a failed execution using stdout, stderr, JUnit, coverage, screenshot, or
trace evidence.

## Input Schema

```json
{
  "type": "object",
  "required": ["test_run", "artifacts"],
  "properties": {
    "test_run": {"type": "object"},
    "artifacts": {"type": "array"},
    "automation_draft": {"type": "object"},
    "test_command": {"type": "object"}
  }
}
```

## Output Schema

```json
{
  "type": "object",
  "required": [
    "classification",
    "confidence",
    "evidence",
    "next_actions",
    "missing_evidence"
  ],
  "properties": {
    "classification": {
      "type": "string",
      "enum": [
        "product_defect",
        "test_script_issue",
        "environment_issue",
        "test_data_issue",
        "dependency_issue",
        "flaky_test",
        "insufficient_evidence"
      ]
    },
    "confidence": {"type": "number"},
    "evidence": {"type": "array"},
    "next_actions": {"type": "array"},
    "missing_evidence": {"type": "array"}
  }
}
```

## Instructions

Return JSON only. Do not include markdown fences in the model output.
Every classification must cite artifact evidence.
If evidence is missing or contradictory, return `insufficient_evidence`.
Do not invent root causes, logs, files, screenshots, or stack traces.

## Failure Output

```json
{
  "error_code": "UNABLE_TO_ANALYZE_FAILURE",
  "message": "Failure evidence is missing or unreadable.",
  "recoverable": true
}
```
