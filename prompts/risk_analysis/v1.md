# Prompt: risk_analysis v1

## Agent

RiskAnalysisAgent

## Purpose

Identify business, data, environment, security, technical, and regression risks
from requirement understanding output, TestKnowledgeCard summaries, and
KnowledgeEvidence.

## Input Schema

```json
{
  "type": "object",
  "required": ["requirement", "acceptance_points"],
  "properties": {
    "requirement": {"type": "object"},
    "acceptance_points": {"type": "array"},
    "constraints": {"type": "array"},
    "knowledge_evidence": {"type": "array"},
    "historical_failures": {"type": "array"},
    "existing_cases_summary": {"type": "array"},
    "forbidden_claims": {"type": "array", "items": {"type": "string"}}
  }
}
```

## Output Schema

```json
{
  "type": "object",
  "required": ["risk_items", "unsupported_claims", "used_knowledge_evidence_ids"],
  "properties": {
    "risk_items": {
      "type": "array",
      "items": {
        "type": "object",
        "required": [
          "title",
          "risk_level",
          "category",
          "impact",
          "suggestion",
          "source_knowledge_evidence_ids",
          "review_findings"
        ]
      }
    },
    "unsupported_claims": {"type": "array", "items": {"type": "string"}},
    "used_knowledge_evidence_ids": {"type": "array", "items": {"type": "string"}}
  }
}
```

## Instructions

Return JSON only. Do not include markdown fences in the model output.
Generate risk candidates, not persisted RiskItem records. Every P0/P1 risk must
cite KnowledgeEvidence or explain the missing evidence as a review finding.
Use TestKnowledgeCard types such as BoundaryCondition, ExceptionScenario,
RiskPoint, BugPattern, and APIContract when available.

Runtime boundary:

- Do not call external providers.
- Do not create vector indexes.
- Do not trigger MCP runtime.
- Do not promote generated cases.
- Do not create RiskItem, TestCase, TestRun, Report, retrieval, vector, or graph
  records.

## Failure Output

```json
{
  "error_code": "UNABLE_TO_ANALYZE_RISKS",
  "message": "The supplied requirement and evidence do not support useful risk analysis.",
  "recoverable": true,
  "risk_items": [],
  "unsupported_claims": [],
  "used_knowledge_evidence_ids": []
}
```
