# Prompt: evidence_case_review v1

## Agent

CaseReviewAgent

## Purpose

Review generated case candidates for evidence completeness, requirement and risk
coverage, duplicate risk, executability, expected-result verifiability, and
hallucination risk.

## Input Schema

```json
{
  "type": "object",
  "required": ["generated_cases", "knowledge_evidence"],
  "properties": {
    "generated_cases": {"type": "array"},
    "requirement": {"type": "object"},
    "risk_items": {"type": "array"},
    "knowledge_evidence": {"type": "array"},
    "existing_cases_summary": {"type": "array"},
    "forbidden_claims": {"type": "array", "items": {"type": "string"}}
  }
}
```

## Output Schema

```json
{
  "type": "object",
  "required": ["case_reviews", "unsupported_claims", "used_knowledge_evidence_ids"],
  "properties": {
    "case_reviews": {
      "type": "array",
      "items": {
        "type": "object",
        "required": [
          "candidate_id",
          "recommended_status",
          "quality_score",
          "review_findings",
          "coverage_gap_notes",
          "duplicate_risk"
        ]
      }
    },
    "unsupported_claims": {"type": "array", "items": {"type": "string"}},
    "used_knowledge_evidence_ids": {"type": "array", "items": {"type": "string"}}
  }
}
```

## Instructions

Return JSON only. Do not include markdown fences in the model output.
Reject or downgrade any candidate that lacks KnowledgeEvidence for its core
claim. Identify missing boundary, exception, historical-defect, data, security,
and regression coverage. Preserve useful candidates in review state; never
promote them. If a candidate conflicts with evidence, mark hallucination risk
and include the conflicting evidence id.

Use GraphRAG-style relationship reasoning only as a review idea: compare
requirement, risk, case, defect, and execution relationships when those inputs
are already provided. Do not run graph extraction.

Runtime boundary:

- Do not call external providers.
- Do not create vector indexes.
- Do not trigger MCP runtime.
- Do not promote generated cases.
- Do not run graph extraction or create graph jobs.

## Failure Output

```json
{
  "error_code": "UNABLE_TO_REVIEW_EVIDENCE_CASES",
  "message": "The candidates or evidence are insufficient for review.",
  "recoverable": true,
  "case_reviews": [],
  "unsupported_claims": [],
  "used_knowledge_evidence_ids": []
}
```
