# Prompt: requirement_review v1

## Agent

RequirementReviewAgent

## Purpose

Generate a six-dimensional requirement review from a requirement and optional
local ContextArtifacts.

## Input Schema

```json
{
  "type": "object",
  "required": ["requirement", "use_knowledge", "context_artifact_ids"],
  "properties": {
    "requirement": {"type": "string"},
    "module_name": {"type": "string"},
    "target_test_types": {"type": "array", "items": {"type": "string"}},
    "use_knowledge": {"type": "boolean"},
    "context_artifact_ids": {"type": "array", "items": {"type": "string"}},
    "context_manifest": {"type": "array"}
  }
}
```
## Output Schema

```json
{
  "type": "object",
  "required": [
    "scores",
    "issues",
    "clarification_questions",
    "test_design_notes",
    "used_knowledge",
    "used_context_artifact_ids"
  ],
  "properties": {
    "scores": {
      "type": "object",
      "required": [
        "completeness",
        "clarity",
        "consistency",
        "testability",
        "feasibility",
        "logic",
        "overall"
      ]
    },
    "issues": {"type": "array"},
    "clarification_questions": {"type": "array"},
    "test_design_notes": {"type": "array"},
    "used_knowledge": {"type": "boolean"},
    "used_context_artifact_ids": {"type": "array", "items": {"type": "string"}}
  }
}
```

## Instructions

Return JSON only. Do not include markdown fences in the model output.
Score each dimension from 0 to 100.
When `use_knowledge` is false, do not claim external knowledge evidence.
If local context is supplied, include the exact ids in `used_context_artifact_ids`.
Every issue must include `dimension`, `severity`, `evidence`, and `suggestion`.

## Failure Output

```json
{
  "error_code": "UNABLE_TO_REVIEW_REQUIREMENT",
  "message": "The requirement could not be reviewed with the provided input.",
  "recoverable": true
}
```
