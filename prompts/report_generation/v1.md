# Prompt: report_generation v1

## Agent

ReportAgent

## Purpose

Generate an evidence-backed Markdown, HTML, or JSON report for a completed test
workflow, including CI/CD quality reports.

## Input Schema

```json
{
  "type": "object",
  "required": ["report_type", "related_entity", "evidence"],
  "properties": {
    "report_type": {"type": "string"},
    "related_entity": {"type": "object"},
    "evidence": {"type": "array"},
    "quality_gate_decision": {"type": "object"},
    "test_runs": {"type": "array"},
    "failure_analysis": {"type": "array"}
  }
}
```

## Output Schema

```json
{
  "type": "object",
  "required": ["conclusion", "summary", "metrics", "evidence_artifact_ids", "next_actions"],
  "properties": {
    "conclusion": {"type": "string"},
    "summary": {"type": "string"},
    "metrics": {"type": "object"},
    "evidence_artifact_ids": {"type": "array"},
    "next_actions": {"type": "array"}
  }
}
```

## Instructions

Return JSON only. Do not include markdown fences in the model output.
Every conclusion must cite artifact ids or structured metrics. Do not mark a
report as `passed` when required evidence is missing. For CI/CD quality reports,
align the report conclusion with the latest QualityGateDecision unless evidence
is missing; missing evidence must produce `insufficient_evidence` or
`needs_attention`.

## Failure Output

```json
{
  "error_code": "UNABLE_TO_GENERATE_REPORT",
  "message": "Required evidence artifacts are missing or unreadable.",
  "recoverable": true
}
```
