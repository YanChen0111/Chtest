# Prompt: knowledge_card_extraction v1

## Agent

KnowledgeIngestionAgent

## Purpose

Extract reviewable TestKnowledgeCard candidates from requirement text, API notes,
defect notes, reviewed cases, or execution evidence. The output is draft
knowledge only; it must preserve source traceability and prompt-safety metadata.

  "properties": {
    "project_id": {"type": "string"},

      "type": "array",

        "type": "object",

## Instructions

Return JSON only. Do not include markdown fences in the model output.
Extract small, test-specific TestKnowledgeCard candidates, not generic document
summaries. Prefer stable section paths or source spans, borrowing the
tree/section traceability idea from PageIndex-style document navigation. Keep
every claim tied to source text. Mark uncertain or conflicting claims in
review_findings instead of smoothing them over.

KnowledgeEvidence may be used only as already-supplied local context; this
prompt must not retrieve it.

Runtime boundary:

- Do not call external providers.
- Do not create vector indexes.
- Do not trigger MCP runtime.
- Do not promote generated cases.
- Do not create retrieval jobs, graph jobs, TestCase records, TestRun records,
  reports, or provider configs.

## Failure Output

  "error_code": "UNABLE_TO_EXTRACT_KNOWLEDGE_CARDS",
  "message": "The supplied artifacts do not contain enough prompt-eligible testing knowledge.",
  "recoverable": true,
  "used_knowledge_evidence_ids": [],
  "unsupported_claims": []
}
