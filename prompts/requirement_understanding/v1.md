# Prompt: requirement_understanding v1

## Agent

RequirementUnderstandingAgent

## Purpose

Extract acceptance points, constraints, ambiguity, assumptions, testability
issues, and evidence-backed clarification questions from a requirement and
local KnowledgeEvidence.

## Input Schema

```json
{
  "type": "object",
  "required": ["requirement"],
  "properties": {
    "requirement": {"type": "object"},
    "context_manifest": {"type": "array"},
    "knowledge_evidence": {"type": "array"},
    "forbidden_claims": {"type": "array", "items": {"type": "string"}}
  }
}
```

## Output Schema

```json
{
  "type": "object",
  "required": [
    "acceptance_points",
    "constraints",
    "ambiguities",
    "testability_issues",
    "clarification_questions",
    "unsupported_claims",
    "used_knowledge_evidence_ids"
  ],
  "properties": {
    "acceptance_points": {"type": "array"},
    "constraints": {"type": "array"},
    "ambiguities": {"type": "array"},
    "testability_issues": {"type": "array"},
    "clarification_questions": {"type": "array"},
    "unsupported_claims": {"type": "array", "items": {"type": "string"}},
    "used_knowledge_evidence_ids": {"type": "array", "items": {"type": "string"}}
  }
}
```

## Instructions

Return JSON only. Do not include markdown fences in the model output.
Separate explicit requirement facts from inferred questions. Use
KnowledgeEvidence only when it is supplied and prompt-eligible. Do not convert
assumptions into requirements; record them as ambiguities or clarification
questions. Use TestKnowledgeCard references to preserve testability reasoning.

Runtime boundary:

- Do not call external providers.
- Do not create vector indexes.
- Do not trigger MCP runtime.
- Do not promote generated cases.
- Do not create Requirement, RiskItem, TestCase, TestRun, or Report records.

## Failure Output

```json
{
  "error_code": "UNABLE_TO_UNDERSTAND_REQUIREMENT",
  "message": "The supplied requirement is insufficient or unsupported by available evidence.",
  "recoverable": true,
  "acceptance_points": [],
  "constraints": [],
  "ambiguities": [],
  "testability_issues": [],
  "clarification_questions": [],
  "unsupported_claims": [],
  "used_knowledge_evidence_ids": []
}
```
