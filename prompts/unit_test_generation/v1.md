# Prompt: unit_test_generation v1

## Agent

UnitTestAgent

## Purpose

Generate a reviewable UnitTestPatch for a local CI/CD diff. The patch must cover
changed source behavior and remain limited to test directories.

## Input Schema

```json
{
  "type": "object",
  "required": ["cicd_run", "changed_files", "risk_analysis"],
  "properties": {
    "cicd_run": {"type": "object"},
    "changed_files": {"type": "array"},
    "risk_analysis": {"type": "object"},
    "target_framework": {"type": "string", "enum": ["pytest", "jest", "vitest"]},
    "existing_tests": {"type": "array"}
  }
}
```

## Output Schema

```json
{
  "type": "object",
  "required": ["patch", "target_framework", "test_intent", "coverage_target", "risk_notes"],
  "properties": {
    "patch": {"type": "string"},
    "target_framework": {"type": "string"},
    "test_intent": {"type": "string"},
    "coverage_target": {"type": "array"},
    "risk_notes": {"type": "array"}
  }
}
```

## Instructions

Return JSON only. Do not include markdown fences in the model output.
Generate a unified diff that modifies test files only. Do not modify product
source, configuration, migrations, generated files, or lockfiles. Do not add
`skip`, `xfail`, broad mocks, or assertions that hide the changed behavior.
Explain the branch or risk covered by the test in `test_intent` and
`coverage_target`.

## Failure Output

```json
{
  "error_code": "UNABLE_TO_GENERATE_UNIT_TEST_PATCH",
  "message": "The diff or test context is insufficient to generate a safe test-only patch.",
  "recoverable": true
}
```
