# Prompt: evidence_case_generation v1

## Agent

CaseGenerationAgent

## Purpose

Generate structured test case candidates from requirements, risks, approved
TestKnowledgeCards, and normalized KnowledgeEvidence. The output is review
material only and must explain why each case exists.

## Input Schema

```json
{
  "type": "object",
  "required": ["requirement", "target_test_types", "knowledge_evidence"],
  "properties": {
    "requirement": {"type": "object"},
    "requirement_review": {"type": "object"},
    "risk_items": {"type": "array"},
    "target_test_types": {"type": "array", "items": {"type": "string"}},
    "approved_knowledge_cards": {"type": "array"},
    "knowledge_evidence": {"type": "array"},
    "existing_cases_summary": {"type": "array"},
    "forbidden_claims": {"type": "array", "items": {"type": "string"}}
  }
}
```

## Output Schema

```json
{
  "type": "object",
  "required": ["cases", "unsupported_claims", "used_knowledge_evidence_ids"],
  "properties": {
    "cases": {
      "type": "array",
      "items": {
        "type": "object",
        "required": [
          "title",
          "priority",
          "test_type",
          "steps",
          "expected_results",
          "covered_requirement_ids",
          "covered_risk_ids",
          "source_knowledge_evidence_ids",
          "generation_reason",
          "automation_readiness",
          "quality_score",
          "review_findings",
          "coverage_gap_notes"
        ]
      }
    },
    "unsupported_claims": {"type": "array", "items": {"type": "string"}},
    "used_knowledge_evidence_ids": {"type": "array", "items": {"type": "string"}}
  }
}
```

## Instructions

Return JSON only. Do not include markdown fences in the model output.
Generate candidate cases, not official TestCase records. Use KnowledgeEvidence
only when it is prompt-eligible and relevant. Each P0/P1 case must cite at least
one source_knowledge_evidence_id unless the output explicitly records a coverage
gap. Use test-design heuristics from professional testing practice: equivalence
partitioning, boundary values, state transitions, exception paths, regression
history, and risk-based priority.

Treat awesome-llm-apps-style agent decomposition as a reference pattern: keep
this prompt focused on generation and leave review, dedup, execution, and
feedback to their own agents.

Runtime boundary:

- Do not call external providers.
- Do not create vector indexes.
- Do not trigger MCP runtime.
- Do not promote generated cases.

## Failure Output

```json
{
  "error_code": "UNABLE_TO_GENERATE_EVIDENCE_CASES",
  "message": "No prompt-eligible KnowledgeEvidence supports useful case generation.",
  "recoverable": true,
  "cases": [],
  "unsupported_claims": [],
  "used_knowledge_evidence_ids": []
}
```
